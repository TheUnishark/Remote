ProxyBlocker Terms

Definitions

(1) Player: Represents a user account in the video-game Broke Protocol.
(2) Proxy: Represents one or multiple intermediate server between the user and the Broke Protocol server. It includes but is not limited to VPN services.
(3) IP: Represents the account's public Internet Protocol address such as an IPv4 or IPv6.
(4) ProxyBlocker API: Represents the service used to validate players IP.


Service Description

ProxyBlocker is a service by TheUnishark that aims to prevent players from using proxies to play in a Broke Protocol server.

Players who connects to a Broke Protocol server with this plugin will be asked if they allow to send their information through this server. They have to agree in order to be able to play the server, otherwise they will not be allowed to perform any in-game interaction in the server.

ProxyBlocker reserve themselves the right to manually block or unblock any IP if they think it's necessary.


User Data

Conditions:

This service will reject a connection to the server if they do not accept the terms or if their IP is detected as a proxy IP.

If a player's IP is unable to be verified by the ProxyBlocker API, the player will be allowed to play the Broke Protocol server and will tell the administration of the server this player is playing without verification.

A player can skip all verifications if their account have the permission pb.bypass.


Usage:

This service sends the player's IP to third parties to verify if it's detected as a proxy. This IP is saved in a database to make the process faster.

This service will maintain the confidentiality of the player's IP address and will not share it with any other third-party organizations or individuals unless required by law.

This service will ensure the security of the player's IP address and will take all necessary measures to prevent unauthorized access or misuse.

This service will comply with all applicable laws and regulations regarding the handling and processing of user data.

Any use of the ProxyBlocker API without authorization will result in legal consequences.


Third parties:

ProxyBlocker takes all necessary measures to protect your data. However, we use third party services and we are not responsible of how they manage data. For further information or request, please contact them.
Note that the only information sent to the third parties are your IP address, and the detection state of it.
